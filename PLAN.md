## Глобальные стандарты UI (обязательно везде)

### Формат даты
- **Везде отображать даты в формате `DD-MM-YYYY`** (например `22-06-2026`)
- Хранение в БД: ISO `YYYY-MM-DD`; `<input type="date">` тоже принимает `YYYY-MM-DD`
- Конвертация для отображения: Jinja2-фильтр `datefmt` в `app.py` (`{{ val | datefmt }}`)

### Выпадающий список филиалов
- **Везде использовать мультивыбор с галочками и группами** (Bootstrap dropdown + checkboxes)
- Паттерн: кнопка-dropdown показывает «Все филиалы» / «Выбрано: N»
- Внутри: кнопки групп филиалов (если есть) + «Все филиалы» чекбокс + отдельные филиалы
- JS-функции: `onBranchCbChange()`, `toggleAllBranches()`, `selectBranchGroup(ids)`
- Параметр формы: `branch_ids` (multiple), принимается через `request.args.getlist('branch_ids')`
- Готовые примеры: `expenses_report.html`, `purchases.html`

### Выпадающий список контрагентов
- **С поиском внутри dropdown и множественным выбором галочками**
- Поле поиска фильтрует список контрагентов прямо внутри dropdown (client-side)
- JS-функции: `filterSuppliers()`, `onSuppCbChange()`, `toggleAllSuppliers()`
- Готовый пример: `purchases.html`

### Контрагенты — единая база
- Единая таблица `contractors` (раздел Банк) — источник истины для всех контрагентов/поставщиков
- При импорте накладных: сопоставлять с `contractors`, новые автоматически добавлять туда
- Дублировать в `purchase_suppliers` для фильтров накладных
