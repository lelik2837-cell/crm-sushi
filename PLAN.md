### 26. ИНН контрагентов

**БД:**
- Новая колонка `inn TEXT` в таблице `contractors` (миграция автоматическая)
- Индекс `idx_contractors_inn` для быстрого поиска

**Сбербанк API (`sber_api.py`):**
- `parse_transactions` теперь извлекает ИНН из `rurTransfer.payeeInn` (для расходов) и `rurTransfer.payerInn` (для доходов)
- Для карточных покупок (operationCode 17) ИНН пропускается — `payeeInn` там ИНН банка-эквайера

**CSV-загрузка:**
- `_map_csv_columns` добавлен паттерн `INN_P` для колонок `ИНН`, `ИНН контрагента` и т.д.
- `_parse_bank_csv` извлекает ИНН из найденной колонки

**Логика матчинга `_match_contractors` (приоритет по убыванию):**
1. По ИНН — `WHERE inn=?` — если найден, привязывается и пропускает остальные шаги
2. По ключевым словам против поля `counterparty` (только, не description!)
3. Если матч по ключевым словам — ИНН из транзакции записывается контрагенту если у него пусто
4. Расход по карте филиала — контрагент не создаётся
5. Новый контрагент — сначала ищем по ИНН, потом по имени; создаём с ИНН если не нашли
- ИНН никогда не перезаписывается если уже заполнен (`AND (inn IS NULL OR inn="")`)

**UI `/bank/?tab=contractors`:**
- Новая колонка «ИНН» в таблице контрагентов
- Поле ИНН в модалках добавления и редактирования

**Маршруты:**
- `bank_contractor_add` и `bank_contractor_edit` принимают и сохраняют поле `inn`

### 27. Исправление ложного матчинга контрагентов

**Проблема:** ООО "КОМПАНИЯ ВОДА" распознавалась как СОУСПАК, ООО
- Причина 1: текст для поиска включал `description` (назначение платежа) — слово «ООО» из ключевых слов СОУСПАК совпадало с «ООО» в поле description
- Причина 2: ключевые слова `"СОУСПАК, ООО"` разбивались по запятой → `["соуспак", "ооо"]`, и `"ооо"` находилось в тексте

**Исправление в `_match_contractors`:**
- Матчинг теперь только против поля `counterparty`, не `description + counterparty`
- Если ключевые слова не заданы вручную (`keywords = ''`) → используется имя контрагента целиком как одна фраза (не разбивается по запятой)
- Если ключевые слова заданы (`keywords` заполнен) → разбиваются по запятой как прежде
