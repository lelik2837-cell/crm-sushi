# CRM Суши — что сделано

## Сессия 2026-06-23

### 1. Детальная страница сотрудника из отчёта ЗП
- В таблице зарплатного отчёта имя сотрудника стало ссылкой (открывается в новой вкладке)
- Новая страница `/reports/employee/<id>` — все смены сотрудника за период
- Показывает: дату, филиал, должность, часы, базу/бонус/штраф, итого, статус выплаты
- Кнопка перехода на карточку каждой смены

### 2. Несколько должностей и ставок в карточке сотрудника
- Новая таблица `employee_roles` в БД
- В модальном окне редактирования сотрудника раздел «Доп. должности и ставки»
- Можно добавить любую должность с отдельной ставкой (для курьера — ₽/км и ₽/заказ)
- Удаление доп. должности крестиком

### 3. Удаление импортированных таблиц смен
- Новая таблица `import_batches` + колонка `import_batch_id` в `shifts`
- Каждый импорт файла Excel создаёт batch-запись
- На странице `/excel-import` внизу — история импортов с датой, файлом, кол-вом смен
- Кнопка «Удалить» — каскадно удаляет смены, расходы, записи ЗП и выплаты

### 4. Увольнение и восстановление сотрудников
- Новые колонки в `employees`: `is_fired`, `fired_at`, `fired_comment`
- В карточке сотрудника кнопка «Уволить» с полем причины
- Уволенные убраны из основного списка → отдельная таблица внизу страницы
- Кнопка «Восстановить» возвращает сотрудника в активные

### 5. Отчёт по движению наличных
- Новый маршрут `/report/cash-flow` в меню Отчёты
- По дням: утром в кассе, выручка нал, расходы нал, ФОТ выплачен, вечером факт в кассе
- Разворот дня — таблица по каждому филиалу с кнопкой перехода на смену
- Фильтры по периоду и филиалу

### 7. Удаление смен (только владелец)
- Новый маршрут `POST /shift/<id>/delete` с декоратором `@owner_required`
- Каскадное удаление: `salary_payments` → `employee_shifts` → `expenses` → `taxi_trip_employees` → `taxi_trips` → `shift_revenue` → `change_log` → `shifts`
- На странице смены красная кнопка «🗑 Удалить смену» видна только владельцу, требует подтверждения
- После удаления — редирект на архив смен

### 8. Импорт накладных из Excel
- Новые маршруты: `GET/POST /purchases/import-excel` и `POST /purchases/import-excel/confirm`
- Ссылка «Импорт накладных» в меню (только для владельца)
- Новый шаблон `purchases_import_excel.html` с 2-шаговым процессом
- **Шаг 1:** загрузка файла `.xls` или `.xlsx`
- **Шаг 2:** маппинг + предпросмотр:
  - Филиалы (из файла → выпадающий список CRM)
  - Контрагенты (текстовые поля, предзаполнены, можно переименовать)
  - Плательщики (галочки — выбрать чьи накладные загружать)
  - Предпросмотр первых 15 строк
- Читаемые колонки: 2-дата, 4-накладная, 9-филиал, 11-контрагент, 14-плательщик, 18-сумма с НДС
- Встроенный парсер BIFF8 для `.xls` (xlrd не справляется с файлами этой системы учёта из-за ошибки OLE-контейнера `seen[3]==4`)
- Добавлена колонка `payer TEXT` в таблицу `purchases`

### 9. Дедупликация накладных при повторном импорте
- Добавлена колонка `import_hash TEXT` в `purchases` + уникальный индекс
- Хэш = MD5(дата|номер_накладной|контрагент_из_файла|сумма)
- При загрузке файла показывается: «X уже есть в базе — будут пропущены»
- Кнопка импорта показывает кол-во новых строк
- Flash после импорта: «Импортировано X, пропущено дублей: Y»

## Глобальные стандарты UI (обязательно везде)

### Формат даты
- **Везде отображать даты в формате `DD-MM-YYYY`** (например `22-06-2026`)
- Хранение в БД: ISO `YYYY-MM-DD`; `<input type="date">` тоже принимает `YYYY-MM-DD`
- Конвертация для отображения: Jinja2-фильтр `datefmt` в `app.py` (`{{ val | datefmt }}`)

### Выпадающий список филиалов
- **Везде использовать мультивыбор с галочками и группами** (Bootstrap dropdown + checkboxes)
- Паттерн: кнопка-dropdown показывает «Все филиалы» / «Выбрано: N»
- Внутри: кнопки групп филиалов (если есть) + «Все филиалы» чекбокс + отдельные филиалы
- JS-функции: `onBranchCbChange()`, `toggleAllBranches()`, `selectBranchGroup(ids)`
- Параметр формы: `branch_ids` (multiple), принимается через `request.args.getlist('branch_ids')`
- Готовые примеры: `expenses_report.html`, `purchases.html`

### Выпадающий список контрагентов
- **С поиском внутри dropdown и множественным выбором галочками**
- Поле поиска фильтрует список контрагентов прямо внутри dropdown (client-side)
- JS-функции: `filterSuppliers()`, `onSuppCbChange()`, `toggleAllSuppliers()`
- Готовый пример: `purchases.html`

### Контрагенты — единая база
- Единая таблица `contractors` (раздел Банк) — источник истины для всех контрагентов/поставщиков
- При импорте накладных: сопоставлять с `contractors`, новые автоматически добавлять туда
- Дублировать в `purchase_suppliers` для фильтров накладных
